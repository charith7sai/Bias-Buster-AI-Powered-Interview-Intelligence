# Project Architecture: Bias Buster AI-Powered Multi-Tier System

This project architecture shows the procedure followed for classification, starting from input to final prediction.

The **Bias Buster** system is architected as a modern, cloud-native application that leverages a decoupled multi-tier pattern to provide a seamless, AI-driven recruitment experience. At its core, the system separates the user interface and presentation logic from the heavy-duty generative AI and data persistence layers. This separation of concerns allows for a highly responsive frontend built on **Next.js 15**, while complex behavioral analysis and bias detection tasks are offloaded to an independent intelligence tier orchestrated by **Genkit**. By using this distributed approach, the system achieves a balance between high-performance client-side interactivity and robust server-side computational depth, ensuring that the heavy processing of transcripts does not compromise the fluidity of the user experience.

The presentation layer is constructed using **React 19** and the **Next.js App Router**, which facilitates efficient server-side rendering and client-side hydration. Styling is handled by **Tailwind CSS** and **ShadCN UI**, ensuring a consistent, accessible, and professional user interface that adheres to modern web standards. This layer communicates with the backend via the **Firebase Client SDK**, which provides a real-time bridge for data synchronization. The use of optimistic UI updates ensures that recruiters see immediate feedback during the interview upload and analysis process, while the system handles the asynchronous nature of AI processing in the background, providing a "live" feel to the data as it populates the dashboard.

The intelligence tier represents the primary value proposition of the Bias Buster project, utilizing the **Genkit** framework to orchestrate sophisticated workflows with Large Language Models (LLMs). When a recruiter submits an interview, the system triggers a specialized **Genkit Flow** that manages the structured extraction of **STAR** (Situation, Task, Action, Result) evidence. This tier does not merely summarize text; it performs a deep cognitive audit of the dialogue to identify subtle indicators of interviewer bias. By defining these processes as structured server-side actions, the system ensures that sensitive analysis logic is centralized, secure, and powered by the advanced reasoning capabilities of **Gemini Pro**, delivering insights that are both deterministic and highly nuanced.

For persistence and security, the architecture relies on a **Backend-as-a-Service (BaaS)** model provided by **Firebase**. **Cloud Firestore** serves as the primary NoSQL database, with a hierarchical data structure designed for scalability and strict user isolation. Security is enforced at the database level through granular **Firestore Security Rules**, which validate that every read or write operation is authorized based on the user's unique authentication token. This multi-layered security approach, combined with **Firebase Authentication**, ensures that confidential interview data and candidate assessments remain strictly private, meeting enterprise-grade standards for data protection and auditability.
